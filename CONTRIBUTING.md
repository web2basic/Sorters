# Contributing to Sorters

Thank you for your interest in contributing to Sorters! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Sorters.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Submit a pull request

## Development Setup

```bash
# Install dependencies
npm install

# Run tests
npm test

# Start local devnet
npm run devnet
```

## Code Style

- Follow existing code style
- Use meaningful variable names
- Add comments for complex logic
- Keep functions focused and small

## Commit Messages

Use clear, descriptive commit messages:

```
feat: Add note search functionality
fix: Resolve tag limit validation issue
docs: Update API documentation
test: Add tests for note deletion
```

## Pull Request Process

1. Update documentation if needed
2. Add tests for new features
3. Ensure all tests pass
4. Update CHANGELOG.md
5. Request review from maintainers

## Testing

- Write tests for all new features
- Ensure existing tests still pass
- Test on both testnet and local devnet

## Questions?

Open an issue or reach out to the maintainers.
